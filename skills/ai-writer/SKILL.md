# AI Writer - 交互式写作助手

## 触发条件

当用户表达以下意图时激活此技能：
- "写一篇文章"
- "帮我写..."
- "创作内容"
- "生成文章"
- "写关于...的内容"

## 工作流程

### 第一步：收集信息

使用 ask 工具向用户询问以下信息：

1. **文章标题**：你想写什么主题？
2. **写作类型**：文章类型（博客文章、学术论文、故事、新闻报道、产品描述、邮件等）
3. **写作风格**：期望的风格（正式、轻松、幽默、专业、诗意、简洁等）
4. **语言选择**：中文还是英文？

示例询问方式：
```
我来帮你写这篇文章！请告诉我：
1. 文章标题或主题是什么？
2. 这是什么类型的文章？（如：博客、论文、故事、新闻、产品描述等）
3. 你希望用什么风格？（如：正式、轻松、幽默、专业等）
4. 用中文还是英文写作？
```

### 第二步：生成文章

根据用户提供的信息，使用你的写作能力生成文章：

- **结构**：根据写作类型选择合适的结构（如博客用引言 - 正文 - 结论，故事用起承转合）
- **风格**：严格遵守用户指定的风格
- **语言**：使用用户选择的语言
- **长度**：根据主题和类型决定合适长度，或询问用户期望长度

### 第三步：提供修改建议

生成后，主动提供：
- 是否需要调整语气或风格？
- 是否需要扩展或精简某些部分？
- 是否需要重新组织内容结构？

## 写作类型指南

| 类型 | 结构建议 |
|------|----------|
| 博客文章 | 吸引人的标题、引言、分点正文、总结/行动号召 |
| 学术论文 | 摘要、引言、文献综述、方法、结果、讨论、结论 |
| 故事 | 开端、发展、高潮、结局 |
| 新闻报道 | 倒金字塔结构：最重要信息在前 |
| 产品描述 | 特点、优势、规格、使用场景 |
| 邮件 | 主题行、问候、正文、结束语 |

## 写作风格指南

- **正式**：使用规范语法，避免缩写，用词精准
- **轻松**：口语化表达，可以使用缩写
- **幽默**：适当使用修辞、双关、轻松的比喻
- **专业**：使用行业术语，数据驱动
- **诗意**：注重韵律、意象、情感表达
- **简洁**：直奔主题，避免冗余

## 示例

### 示例 1
用户："我想写一篇关于 AI 的文章"
操作：启动信息收集流程，询问标题、类型、风格、语言

### 示例 2
用户提供完整信息后：
- 标题："AI 如何改变我们的工作方式"
- 类型：博客文章
- 风格：专业但易懂
- 语言：中文

输出：一篇结构完整、观点清晰的博客文章
