Understanding React Server Components: A Deep Dive

📖 1,234 词 ⏱️ 约 8 分钟 🎤 128 人跟读

React Server Components represent a paradigm shift in how we build React applications. They allow us to write components that render exclusively on the server, reducing the amount of JavaScript sent to the client and improving initial page load performance.

React 服务器组件代表了构建 React 应用程序的范式转变。

Traditional React components run on both the server and the client, which means they need to include all the code necessary for hydration. Server Components, on the other hand, never run on the client. This allows them to import and use server-side only dependencies like database clients directly.

ExampleComponent.tsx 
import db from './database'
import { Note } from './Note'

export async function NoteList({ playerId }) {
  const notes = await db.notes(playerId).all()
  return <ul>{notes.map(note => <Note key={note.id} note={note} />)}
}

The key insight is that Server Components send UI as a serialized format to the client. This format is not HTML or JSON, but a custom wire format designed specifically for React. The client receives this format and seamlessly integrates it with existing client components.

One of the most powerful features is automatic code-splitting. When you use a Server Component, its dependencies are automatically split into separate bundles. This happens without any additional configuration from the developer.
