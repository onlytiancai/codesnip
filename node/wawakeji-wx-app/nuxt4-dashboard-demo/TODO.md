 - 文章管理页面 /admin/articles 的添加和修改增加按句分割和按段落分割，分割后的句子和段落用json 存储，不存在单独的表里了
      - 按段落分割时，每个段落保存它的原文，译文，音频
      - 按句子分割时，每个句子保存它的原文，译文，音频，音标，其中音标是个数组，要和原句每个单词一一对应
      - 获取翻译，音频的 API 先实现一个假的，后续再接入第三方API
      - 获取句子音标的 API 可以使用本地字典 server/utils/dictionary-db.ts 实现一个
      - 在为段落和句子批量生成翻译，音频，音标时，增加一个进度显示，不要并发，要串行执行，请求从前端依次发起
      - 当用户保存时，做完成性检查，防止有些句子或段落没有对应的翻译，音频或音标
  - 文章显示页 /articles/{slug} 提供3种显示模式，
      - 一种是原文显示模式，就是纯英文，鼠标放在单词上可以弹出翻译
      - 一种是沉浸式翻译模式，既按段落分割显示，一段英文，一段中文，每段末尾有个按钮可以播放本段，也有按钮可以顺序播放所有段落。
      - 一种是逐句跟读模式，每句可以播放，也可以全部顺序播放，是否显示每句的音标只在这种模式可用。
