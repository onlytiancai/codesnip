# 第 9 章 梯度下降 + 训练循环

## 9.1 梯度下降：往最陡的下坡走

**梯度下降**（gradient descent）= 优化的核心算法。

**基本想法**：
- 损失 L 是 W 的函数（一个"山"）
- 梯度 $\partial L / \partial W$ = 山坡的"陡度"和"方向"
- 沿负梯度方向走一步 → L 减小 → 优化

**更新公式**：
$$
W \leftarrow W - \alpha \cdot \frac{\partial L}{\partial W}
$$

- `α`（alpha）= 学习率 = 步子大小
- 梯度 $\frac{\partial L}{\partial W}$ 由反向传播算出
- 每走一步，W 都更新一次

::: chart caption="梯度下降：沿负梯度方向下山"
![梯度下降](assets/images/ch09_gradient_descent.png)
:::

**几何理解**：
- 想象站在山上，目标是走到谷底（L 最小）
- 当前点的梯度告诉你"哪个方向最陡"
- 朝最陡的反方向（负梯度）走一步
- 重复

---

## 9.2 学习率：步子大小

**学习率 α 的影响**：

| α | 效果 |
|---|---|
| 太小 | 收敛超慢、卡在"半山腰" |
| 太大 | 在最优点附近"荡秋千"、发散 |
| 合适 | 平滑下降到谷底 |

**怎么选 α？**
- 经验值：先试 0.1
- 看 loss 曲线：↓平稳 = 合适；↓震荡 = 太大；↓不动 = 太小
- 现代优化器（Adam）能自适应 α

**学习率调度（learning rate schedule）**：
- 一开始用大 α（快速接近最优点）
- 后面用小 α（精细调整）
- 例：`α = 0.1 / (1 + 0.01·epoch)`

---

## 9.3 训练循环：前向 → 算损失 → 反向 → 更新

**一个完整 epoch 的流程**：

```python
for epoch in range(num_epochs):
    for x_batch, y_batch in dataloader:
        # 1. 前向
        y_pred = model(x_batch)
        # 2. 算损失
        loss = criterion(y_pred, y_batch)
        # 3. 反向
        loss.backward()
        # 4. 更新（梯度下降）
        optimizer.step()
        # 5. 清零梯度
        optimizer.zero_grad()
```

**关键点**：
- 每个 epoch 跑**整个数据集**一遍
- 4 个步骤**缺一不可**（少了任何一步训练都不工作）
- 反向前要 `zero_grad()`，否则梯度会累加

梯度下降更新 $W \leftarrow W - \alpha \cdot \frac{\partial L}{\partial W}$ 中，$\alpha$ 的角色是？
- A: 梯度本身
- B: 步子大小
- C: 损失值

answer: B

> $\alpha$ = 学习率 = 步子大小。控制每次走多远。

::: quiz q9-2 single
训练时 loss 一直是 0.69 不下降（对 2 分类），最可能的原因是？
- A: 学习率太大
- B: 学习率太小
- C: 网络输出完全是 0.5（没学到东西）
- D: 数据集太大

answer: C

> Loss = 0.69 = -log(0.5) = 随机猜测的水平。
> 原因通常是：模型结构错了 / 标签反了 / 训练完全没工作。
> 解决：检查输出、打印中间值、换一个简单模型对比。
:::

::: quiz q9-3 short placeholder="解释 Adam 优化器的好处"
**问题**：现代深度学习几乎都用 **Adam** 优化器（自适应学习率）。它和普通梯度下降相比，主要好在哪？

> 参考答案（不唯一）：
> 普通 SGD 用同一个 α 走所有方向 → 容易在陡的方向震荡、在缓的方向卡住。
> 
> Adam 的改进：
> 1. **自适应 α**：每个参数有自己的学习率（梯度大的参数 → 小 α；梯度小的参数 → 大 α）
> 2. **动量**：累积过去梯度的"惯性"，跳过局部小坑
> 3. **偏差修正**：开始几次更新时修正统计量
> 
> 实际效果：调参简单（默认参数就不错），训练更快、更稳。
:::
